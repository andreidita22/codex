# Fork Docs

This directory contains fork-only documentation.

Use it for materials that describe:

- fork-local feature extraction and hardening plans
- fork-local audits and review notes
- custom module inventories and build/update guidance
- upstream-ingest watchlists specific to fork-owned behavior

The root [`docs/`](../) directory should stay
focused on repo-wide or vanilla-aligned documentation.

Current fork-local families documented here include:

- context maintenance / compaction policy
- agent observability / E-witness
- fork update and release-alignment process
- release-specific post-ingest alignment specs
